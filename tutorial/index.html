Self-healing locator framework

Keep the test flow. Heal only the locator.

For black-box CMS websites where markup changes arrive before anyone updates the automation suite.

Pipeline

Cache first, scan second, generate top 5, validate in order.

1. TargetThe test asks for “login button”.
2. CacheTry stored candidates by confidence.
3. ScanExtract visible components and hashes.
4. GenerateLLM-ready adapter ranks locators.
5. LearnWorking locator updates confidence.
Confidence metrics

A locator cache should behave like evidence, not a string drawer.

Web and mobile

The method is portable, but the scan surface changes.

Website

  • Whole DOM can be scanned.
  • Candidate locators can be ranked page-wide.
  • Cache keys use URL plus target name.

Android / iOS

  • Appium sees the current viewport.
  • Off-screen elements need scroll collection.
  • The flow stays fixed; locator choice adapts.
